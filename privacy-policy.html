Privacy Policy – WiFi HTTP Server Free

Effective Date: November 30, 2025
Developer: ByteCode Creation
Contact: bytecode.creation@gmail.com

1. Introduction

WiFi HTTP Server Free ("the App") is a local file-sharing utility that allows users to start a temporary HTTP server on their Android device and share files with other devices connected to the same WiFi network.

This Privacy Policy explains what data the App collects, how it is used, how it is shared, and your rights. By using the App, you agree to the terms described here.

2. What the App Does

The App:

  • Starts a temporary local HTTP server while running in the foreground
  • Lets you choose a folder to share
  • Allows other devices on the same WiFi network to access the folder via a browser
  • Displays banner and interstitial ads using Google AdMob (free version only)
  • Does not upload or transmit your files to any external server

3. Information the App Collects (Locally Only)

The App itself does NOT collect or transmit personal data to any server. However, to function properly, it processes certain information locally on your device:

1. Local IP Address

  • Used only to display the server URL (e.g., http://192.168.x.x:8080).
  • Not sent outside your device.

2. Selected Folder Path

  • The folder you choose to share is stored locally for server operation.
  • The App only serves files from the selected folder.

3. App Preferences

  • Such as port number, theme, upload toggle, or last selected folder.
  • Stored locally via SharedPreferences.

None of this information is uploaded, stored on remote servers, or shared by the App itself.

4. Information Collected by Third-Party Services (Ads – AdMob)

The free version uses Google AdMob to display ads. AdMob and its advertising partners may collect and process data such as:

  • Device identifiers (including Advertising ID)
  • IP address
  • Approximate location (derived from IP)
  • App interactions and ad impressions
  • Ad clicks and performance data

These are collected directly by Google, not by this App or the developer.

For more details, see Google’s policies: https://policies.google.com/technologies/ads

5. Why Data Is Used

App Data (Local Only)

Used solely for core functionality:

  • Running the local HTTP server
  • Loading your selected folder
  • Remembering your app settings

AdMob Data

Collected and used by Google and its partners for:

  • Delivering ads
  • Ad personalization (if enabled)
  • Frequency capping
  • Fraud prevention
  • Ad measurement

None of this data is processed by the App directly.

6. Storage Access & MANAGE_EXTERNAL_STORAGE

To share files, the App may request storage access. Whenever possible, the App uses Android’s Storage Access Framework (SAF) so that you can choose a folder without granting broad access.

In special cases, if needed for full functionality, the App may request the optional MANAGE_EXTERNAL_STORAGE permission (“All Files Access”). This permission is:

  • Optional
  • Clearly explained before requesting
  • Used only to allow serving files from folders the user chooses
  • Never used to collect personal data

Google Play requires justification for this permission, and the App complies with those requirements.

7. Data Retention and Deletion

  • All data handled by the App is stored locally on your device.
  • The App does not upload any file, folder list, or server data externally.
  • When you uninstall the App, all stored data (preferences and selected folder information) is removed automatically.
  • You may also clear the App’s storage manually from Android settings.

8. User Choices and Rights

Opt-out of Personalized Ads

You may disable personalized advertising at any time:

  • In your Google account ad settings
  • In your device’s “Ads” or “Privacy” settings

Disable Uploads

The App includes a setting to enable or disable file uploads to your device.

Deleting Local Data

You may delete all local data by clearing the App’s storage or uninstalling it.

Request for Help or Information

You may contact the developer at: bytecode.creation@gmail.com

9. Security

  • Your files are never uploaded to any server by the App.
  • The HTTP server works only on your local network.
  • The server stops when the App is closed or minimized (free version).
  • Only the folder you select becomes accessible on the network.

For best security, run the server only on trusted WiFi networks.
